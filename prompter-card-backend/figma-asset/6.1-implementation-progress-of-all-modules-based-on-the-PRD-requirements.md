## 📝 Notes

### Key Business Rules

1. **Single-Tenant** - One business per deployment (no multi-tenancy)
2. **No Subscription Billing** - Not a SaaS product (unless changed later)
3. **NFC + QR Support** - Both card types supported
4. **Games Required** - Every landing page must have a game
5. **Review Tracking** - Click tracking only (Google API limitation)
6. **GDPR Compliance** - Consent required for all sign-ups

### Technical Decisions

1. MongoDB for database (flexible schema for landing page blocks)
2. Redis for cache, sessions, and queues
3. BullMQ for async operations (reports, exports, emails)
4. Generic controllers/services for DRY principle
5. Cache-aside pattern with TTL by data type
6. Rate limiting tiers (auth, public, user, admin)
7. Zod validation (100% endpoint coverage)
8. Winston for structured JSON logging
