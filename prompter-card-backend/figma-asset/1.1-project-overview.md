## Executive Summary

PromoterCard is a modern web platform that replaces traditional flyer-based event promotion with interactive NFC-enabled cards linked to customizable landing pages. The system enables businesses (primarily hospitality venues like beach clubs, bars, and lounges) to deploy promoters with NFC cards that engage potential customers through interactive games, guest list sign-ups, and Google review incentives. The platform provides comprehensive analytics to measure campaign effectiveness and promoter performance.

**Key Value Proposition:** Transform outdated, untrackable flyer marketing into measurable, engaging digital experiences that drive customer acquisition and brand loyalty.

---

## Problem Statement

Traditional event promotion methods (flyers, word-of-mouth) suffer from:
- **Low engagement** - Customers discard flyers without interaction
- **No measurability** - Impossible to track ROI or promoter performance
- **High waste** - Print materials with limited lifespan
- **Zero interactivity** - No immediate engagement mechanism

---

## Solution Overview

PromoterCard provides:
- **Interactive Engagement** - Games (dice roll, spin wheel) incentivize participation
- **Measurable Results** - Real-time analytics on every interaction
- **Data Capture** - Build customer databases for retargeting
- **Reputation Building** - Systematic Google review collection
- **Promoter Accountability** - Track individual promoter performance

## Key Decisions Made

| Decision | Rationale | Impact |
|----------|-----------|--------|
| Single-Tenant Architecture | Simplified scope, faster time to market | No multi-tenancy, easier implementation |
| MongoDB (Not PostgreSQL) | Flexible schema for landing page blocks | Better fit for document-based data |
| Generic Controllers/Services | DRY principle, consistency | Faster development, maintainable code |
| Redis Cache-Aside Pattern | Performance optimization | 10x faster read operations |
| BullMQ for Heavy Operations | Async processing for scale | Non-blocking API responses |
