## 🔧 Technical Specifications

### **Endpoint**:
```
GET /children-business-users/team-members/:memberId
```

### **Auth**:
- Business user (parent/teacher)
- Must verify member belongs to business user

### **Response Structure**:
```json
{
  "success": true,
  "data": {
    "member": {
      "_id": "...",
      "name": "Alax Morgn",
      "email": "alax@example.com",
      "phoneNumber": "1234567890",
      "gender": "male",
      "dob": "2021-12-12",
      "age": 6,
      "address": "USA",
      "profileImage": {...},
      "supportMode": "calm",
      "roleType": "Secondary"
    },
    "tasks": [
      {
        "_id": "...",
        "title": "Complete Math Homework",
        "status": "completed",
        "startTime": "2026-10-12T08:30:00.000Z",
        "description": "...",
        "assignedUserIds": [...],
        "subtasks": [
          {
            "_id": "...",
            "title": "Call with design team",
            "isCompleted": true,
            "completedBy": {...}
          }
        ],
        "taskType": "collaborative",
        "completionPercentage": 100
      }
    ],
    "statistics": {
      "totalTasks": 12,
      "completedTasks": 2,
      "pendingTasks": 10,
      "inProgressTasks": 0
    }
  }
}
```

### **Caching Strategy**:
- Cache key: `children-business-user:member:{memberId}:details`
- TTL: 3 minutes (member details change infrequently)
- Invalidate on task creation/update/deletion

### **Performance Targets**:
- Response time: < 200ms
- Cache hit rate: > 80%
- Database queries: Optimized with indexes
